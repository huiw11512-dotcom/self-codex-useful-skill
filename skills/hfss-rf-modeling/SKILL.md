# HFSS RF Modeling

## Operating Rules

- Inspect the existing AEDT project/script first. Preserve the current useful project unless the user explicitly asks to delete or rebuild, or the geometry is unrecoverable.
- Prefer parameter edits over redraws: line lengths, widths, phase sections, clearance, monopole height/radius, port size/location, and mappings should be variables.
- Before editing, identify the actual feed family: microstrip edge feed, coax feed, via/probe feed, lumped gap test, or full array integration.
- If the user is asking about ports, excitation setup, screenshots, or repeated port errors, read [references/hfss_port_rules.md](references/hfss_port_rules.md).
- If editing Python/VBScript/PyAEDT generation scripts, run `scripts/lint_hfss_ports.py <script>` when possible before simulation.

## Port Policy

Do not use an arbitrary visible rectangular sheet next to the antenna/radiator as a final coax or monopole excitation. A small rectangular lumped-port sheet is only acceptable as a temporary gap-feed test when clearly labeled and when the user agrees.

Use these defaults:

- **Microstrip input**: keep it microstrip unless the user asks otherwise. Use an edge wave port on the board boundary or a lumped port from trace to ground at the microstrip reference plane.
- **Coax-fed monopole/probe**: create inner conductor, dielectric annulus, outer conductor tied to ground, and a wave port on the coax end cross-section. The integration line goes from inner conductor to outer conductor.
- **Via/probe to top radiator**: keep the center conductor continuous through the ground clearance. Ground must have a clearance hole. Do not place the excitation at the top clearance in the integrated model.
- **Butler output to antenna**: after integration, the Butler output is not an external S-port unless the task is `butler_only`. It should connect to the probe/via feeding the antenna.

Hard stops before simulation:

- Port face intersects metal or is buried inside a conductor.
- Wave port is not on an exterior/open boundary or valid terminal cross-section.
- Coax outer conductor is not connected to the reference ground.
- Coax port has only one conductor touching the port face.
- Microstrip input was changed to coax without user approval.
- Radiation boundary is used to hide or compensate for a bad port.

## Workflow

1. Save the current project before risky changes.
2. Make a focused edit in the existing project/script.
3. Validate geometry names, variables, booleans, and port placement.
4. Run a small simulation first: single frequency or coarse sweep.
5. Report S-parameters and phase using exact frequency points, not only screenshots.
6. If results are poor, tune the smallest parameter set that explains the error.

For Butler/OAM work, optimize in this order:

- Port/reference-plane correctness.
- Hybrid/crossover connectivity and isolation.
- Phase-shifter and output length compensation.
- Antenna resonance and probe clearance.
- Full array coupling and element mapping.

## HFSS Optimization Model Rules

- Separate pattern-shaping variables from matching variables. For antennas, radius, height, pitch/turns, ground size, and parasitic metal mostly shape pattern; feed radius, strip width, taper length/width, pads, and probe transitions mostly shape S-parameters.
- Do not optimize a variable unless the model history is bound to that variable. Numeric points generated by Python do not update when an AEDT variable changes unless the point coordinates use expressions or the model is rebuilt.
- Keep coax feeds physical throughout optimization: fixed wave-port cross sections are acceptable, but center conductors must remain connected to the radiator through a real metal transition and the outer conductor must remain tied to ground.
- When optimizing wrapped helices, parameterize the virtual-cylinder wrap rather than using horizontal winding or unrelated polyline shortcuts. Check that the short ring remains closed and that upper-feed pins are not shorted by lower short plates.
- For wide-beam circular polarization targets, constrain both matching and pattern: include `dB(S(...)) <= target` at band-edge points, include circular-polarized gain goals at multiple theta/phi samples across the desired beam, and add a boresight upper limit if the optimizer keeps making a narrow high-gain lobe.
- Treat a sudden loss of S-parameter quality after parameterization as a topology/connection problem first: check feed-link continuity, port naming, short-ring holes, and whether the optimized variables are actually connected to feed pads.
- For stacked L/S quadrifilar helix work, solve and report both bands for every full candidate unless explicitly running a labeled diagnostic. L uses lower ports `L1-L4` over `1.5-1.7 GHz`; S uses upper ports `U1-U4` over `2.0-2.2 GHz`; driven phases are `0/90/180/270`.
- For QHA cardioid beamwidth targets stated in dBic, judge the selected circular-polarized component (`GainLHCP` or `GainRHCP`) over `Theta=-180..180` and multiple phi cuts. Do not claim success from `GainTotal` crossings or from a high-gain narrow lobe crossing 0 dB.
- When using thin wrapped arms for speed, verify that the feed pad, arm facets, and end short are electrically continuous. If sheet/solid contact is uncertain, use low-segment solid wrapped ribbons for topology validation before optimizing dimensions.
- Separate pattern and matching actions: helix radius, height, turns, ground size, and stack spacing shape the cardioid; feed radius, pad size, taper width/length, coax shield extent, and matching patches tune S-parameters. A poor active pattern with very poor S-parameters can be dominated by feed currents rather than QHA currents.

## Reporting

Be direct. State what changed, which AEDT project was saved, whether simulation ran, and the key results at the target frequency. If a port remains temporary, say so explicitly.
